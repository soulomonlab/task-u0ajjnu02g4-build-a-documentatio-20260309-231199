## Database Design for Documentation Automation (update_docs_from_pr)

This design models a documentation tool that ingests PRs (pull requests), maps code changes to doc impact, tracks checklists and follow-up PR recommendations, and records worker jobs that performed analyses. It supports multiple projects/repos, workers (agents), and produces actionable doc tasks and recommendations.

### Tables
- projects (id uuid PK, name, slug, description, created_at, updated_at)
- repositories (id uuid PK, project_id FK → projects.id, name, url, default_branch, created_at, updated_at)
- users (id uuid PK, username, email, display_name, role, created_at)
- pull_requests (id uuid PK, repo_id FK → repositories.id, pr_number int, title, author_id FK → users.id, body text, base_branch, head_branch, state varchar, created_at, merged_at, metadata jsonb)
- pr_files (id bigserial PK, pr_id FK → pull_requests.id, file_path text, change_type varchar, diff TEXT, file_blob_hash text)
- doc_tasks (id uuid PK, pr_id FK → pull_requests.id, task_type varchar, summary text, description text, status varchar, priority int, assigned_to uuid FK → users.id NULL, created_by uuid FK → users.id, created_at, updated_at, metadata jsonb)
- doc_changes (id uuid PK, task_id FK → doc_tasks.id, file_path text, change_description text, impact_level varchar, suggestion text, created_at)
- checklist_templates (id uuid PK, name, description, owner_id FK → users.id, visibility varchar, created_at, updated_at, schema jsonb)
- checklists (id uuid PK, template_id FK → checklist_templates.id NULL, task_id FK → doc_tasks.id NULL, name, status varchar, created_at, updated_at)
- checklist_items (id uuid PK, checklist_id FK → checklists.id, key varchar, label text, required boolean, status varchar, notes text, order int)
