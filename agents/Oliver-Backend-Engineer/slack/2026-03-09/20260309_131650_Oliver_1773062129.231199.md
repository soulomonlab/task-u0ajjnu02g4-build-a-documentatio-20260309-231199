## Database Design for Documentation Tracking (code→docs impact, checklist, follow-up PRs)

Goal: track code PRs, mapped documentation impacts, checklist items, and recommended docs follow-up PRs. Support queries by repo, PR, file, worker/provider, status, and full-text search of recommendations/impact notes.

### Tables
- repos
  - id UUID PK
  - name TEXT NOT NULL
  - remote_url TEXT
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()

- users
  - id UUID PK
  - username TEXT NOT NULL UNIQUE
  - display_name TEXT
  - email TEXT
  - role TEXT CHECK (role IN ('worker','provider','po','bot','other'))
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()

- tasks
  - id UUID PK
  - repo_id UUID REFERENCES repos(id) ON DELETE CASCADE
  - title TEXT
  - description TEXT
  - created_by UUID REFERENCES users(id)
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - status TEXT CHECK (status IN ('open','in_progress','done','archived')) DEFAULT 'open'

- prs (code pull requests)
  - id UUID PK
  - repo_id UUID REFERENCES repos(id) ON DELETE CASCADE
  - pr_number INT NOT NULL
  - title TEXT
  - author_id UUID REFERENCES users(id)
  - url TEXT
  - merged BOOLEAN DEFAULT FALSE
  - merged_at TIMESTAMP WITH TIME ZONE
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - summary JSONB -- aggregated code changes summary (files touched, modules, types of change)
  - UNIQUE(repo_id, pr_number)

- pr_files
  - id UUID PK
  - pr_id UUID REFERENCES prs(id) ON DELETE CASCADE
  - path TEXT NOT NULL
  - change_type TEXT CHECK (change_type IN ('added','modified','deleted','renamed'))
  - diff_summary TEXT
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - UNIQUE(pr_id, path)

- doc_impact_items
  - id UUID PK
  - pr_id UUID
